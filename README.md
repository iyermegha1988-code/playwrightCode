**#Test Structure**
**API**: Contains tests for authentication and backend validations.
**EndToEnd**: Covers complete user journeys, demonstrating interactions with radio buttons, checkboxes, frames, dropdowns, new tabs/windows, and login-to-order flows.
**NetworkRequests**: Handles API interception and response mocking to test network-dependent scenarios.
**ExcelReadWrite**: Implements data-driven testing using Excel for dynamic test data.
